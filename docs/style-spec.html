Style Spec, no detalhe

Espelho vivo do código. Tokens, classes, componentes — extraídos de tailwind.config.ts, index.css, packages/shared e components/canvas/**. Estética canvas-on-paper. Um verde. Geist + Instrument Serif italic.

floating-panel · 12px radius · white/95 + blur snap 16px · easing cubic-bezier(.16,1,.3,1)

1 · Princípios

Decisões fundadoras. Tudo o que segue se ancora aqui.

01
Floating-first

Toda UI flutua sobre o canvas. white/95 + blur + shadow-float. Nada opaco grudado em borda.

02
Uma cor de marca

#C5F432 é o único verde. Selection, focus, active state, accent. Nada secundário.

03
Categoria = cor + forma

Losango pra tráfego, círculo pra relacionamento, etc. Forma reforça categoria além da cor.

04
Família visual

atomic · page · sticky · text. Cada uma com proporção, sombra e edição própria. Não misturar.

05
Mockup-realismo

Pages renderizam browser window (3 dots, URL, wireframe interno). Não retângulo abstrato.

06
Movimento contido

150–280ms · out-expo. Loops decorativos 1–1.6s. Nada de bounce, nada > 400ms em UI funcional.

07
Snap 16px

Grid, posição, padrão de fundo, padding. Tudo múltiplo de 16. Coerência espacial dogma.

08
pt-BR UI, EN código

Labels de produto em português. Tokens, classes, identificadores em inglês.

7 · Motion

Easing canônico cubic-bezier(.16,1,.3,1) (out-expo). 150–280ms em UI funcional. Loops 1–1.6s.

dash · edge
1.6s · linear · infinite
pulse-brand
1.4s · ease-out · infinite
Duração por contexto
  • Entrada de UI (painel, hint, sidebar, bottom-bar): 180–220ms
  • Zoom in/out programático: 200ms · fitView: 280ms
  • Hover de botão / scale 1.05 do ícone: padrão ~150ms ease
  • Loops decorativos (dash, pulse): 1–1.6s
  • Connect-mode opacity transition: 160ms ease
  • Submenu open delay 120ms · close delay 180ms

15 · Anti-padrões

Decisões que parecem boas e quebram a linguagem. Não fazer.

Sombras pesadas

Não usar shadow-lg/xl/2xl do Tailwind. Quebra estética fria do canvas-on-paper. Apenas as 3 canônicas: float · float-lg · sticky.

Acento secundário

Não criar rosa/ciano/roxo "extra" pra destacar. Só brand-300. Hierarquia visual vem de tipografia + peso + zinc, não de cores extras.

Raio arbitrário

Nada de rounded-[7px], rounded-[20px]. Escala discreta: 4 · 6 · 8 · 12 · 14 · 999. Cantos retos proibidos em painéis flutuantes.

Painel opaco

Nunca bg-white 100%. Sempre white/95 + backdrop-blur. Transparência é parte da linguagem.

Ícone fino

Lucide strokeWidth=1 ou 1.5 some no canvas bege. 2 em UI, 2.25 dentro de chip colorido, 2.5 pra "carimbar" (check, overlay).

Texto branco em brand

Texto sobre #C5F432 deve ser zinc-900. Branco falha contraste. Nunca branco.

Animação longa em UI

> 300ms em entrada de menu/painel é lento. Reservado pra loops decorativos (pulse, dash).

transform: scale() no resize

Removido. Tentativas via useInternalNode/ResizeObserver instáveis. Resize só altera bounding box; conteúdo centraliza.

Auto-delete de TextNode vazio

Removido por race condition (closure stale durante onChange rápido deletava nós com texto recém-digitado). Vazio fica como placeholder italic.

Cor hardcoded fora do registro

Tipo → NODE_TYPES[kind].color. Neutro → escala zinc. Nunca hex solto inline.

Wireframe genérico em Page

Cada kind de página tem mockup próprio em PageWireframe. Não cair pra retângulos genéricos por preguiça.

font-display fora de hero

Instrument Serif italic é raro de propósito. Só empty state / floreio. Nunca em body, botão, label.
